## 5. p-queue 替代方案评估

### 5.1 p-queue 能力分析

**p-queue 核心功能**:
- ✅ 优先级队列
- ✅ 并发控制 (concurrency limit)
- ✅ 超时处理
- ✅ 暂停/恢复
- ✅ 事件监听 (idle, active, add)

**CodePilot 需求对比**:

| 需求 | p-queue 支持 | 说明 |
|------|-------------|------|
| 优先级调度 | ✅ | 原生支持 |
| 并发控制 | ✅ | 原生支持 |
| 资源感知 | ❌ | 需要自行实现 |
| 崩溃恢复 | ❌ | 需要自行实现 |
| 任务依赖 | ❌ | 需要自行实现 |

### 5.2 需要补充的功能

**1. 资源感知调度** (1-2 天)
```typescript
class ResourceAwareQueue extends PQueue {
  async add(task, priority) {
    const cpuUsage = await getCPUUsage();
    const memUsage = await getMemoryUsage();

    if (cpuUsage > 80 || memUsage > 80) {
      // 降低并发度
      this.concurrency = Math.max(1, this.concurrency - 1);
    }

    return super.add(task, priority);
  }
}
```

**2. 崩溃恢复** (1-2 天)
```typescript
class PersistentQueue extends PQueue {
  async add(task, priority) {
    // 持久化任务到 SQLite
    await db.insert('tasks', { id, priority, status: 'pending' });

    return super.add(async () => {
      try {
        await task();
        await db.update('tasks', { id, status: 'completed' });
      } catch (e) {
        await db.update('tasks', { id, status: 'failed', error: e });
      }
    }, priority);
  }
}
```

**3. 任务依赖** (2-3 天)
```typescript
class DependencyQueue extends PQueue {
  dependencies = new Map();

  async addWithDeps(task, deps = []) {
    // 等待依赖任务完成
    await Promise.all(deps.map(d => this.dependencies.get(d)));

    const promise = this.add(task);
    this.dependencies.set(task.id, promise);
    return promise;
  }
}
```
