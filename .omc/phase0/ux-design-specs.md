# Phase 0 UX 设计规范

## 目标
为 CodePilot vNext 定义核心用户体验规范，确保性能优化对用户可见且易于理解。

---

## 1. 加载状态规范

### 1.1 解析进度指示器

**场景**: Tree-sitter 增量解析大型项目时

**设计要求**:
- 显示当前解析文件数 / 总文件数
- 显示解析速度（文件/秒）
- 预估剩余时间
- 可取消操作

**UI 元素**:
```
正在解析代码库...
进度: 1,234 / 10,000 文件 (12%)
速度: 45 文件/秒
预计剩余: 3 分 12 秒
[取消]
```

### 1.2 推理状态指示器

**场景**: Claude API 调用时

**设计要求**:
- 显示当前推理任务类型（代码补全/Chat/搜索）
- 显示等待时间
- 区分网络延迟 vs 模型推理时间

**UI 元素**:
```
正在生成代码...
任务: 代码补全
已等待: 1.2 秒
状态: 模型推理中
```

### 1.3 向量检索状态

**场景**: FAISS 向量搜索时

**设计要求**:
- 显示检索到的相关代码片段数量
- 显示检索耗时
- 显示相似度分数范围

**UI 元素**:
```
检索相关代码...
找到 15 个相关片段
耗时: 45ms
相似度: 0.82 - 0.95
```

## 2. 错误提示模板

### 2.1 解析错误

**错误类型**: 文件过大

**友好提示**:
```
⚠️ 文件过大无法解析

文件 "src/generated/large-file.ts" 大小为 15MB，超过 10MB 限制。

建议:
• 将大文件拆分为多个小文件
• 排除自动生成的文件（在 .gitignore 中添加）
• 使用 Tree-sitter 降级模式（仅解析函数签名）

[查看详情] [忽略此文件]
```

**错误类型**: AST 深度超限

**友好提示**:
```
⚠️ 代码嵌套层级过深

文件 "src/utils/nested.ts" 的 AST 深度为 520 层，超过 500 层限制。

建议:
• 重构代码，减少嵌套层级
• 使用提前返回（early return）模式
• 拆分复杂函数

[查看详情] [跳过此文件]
```

### 2.2 推理错误

**错误类型**: API 调用失败

**友好提示**:
```
❌ 无法连接到 Claude API

错误: 网络超时（5 秒）

可能原因:
• 网络连接不稳定
• API 服务暂时不可用
• 防火墙阻止了请求

建议:
• 检查网络连接
• 稍后重试
• 查看 API 状态页面

[重试] [查看日志] [切换到离线模式]
```

### 2.3 向量检索错误

**错误类型**: 索引损坏

**友好提示**:
```
⚠️ 向量索引需要重建

检测到索引文件损坏或版本不兼容。

影响:
• 代码搜索功能暂时不可用
• Chat 功能可能受限

建议操作:
• 重建索引（约需 5-10 分钟）
• 继续使用其他功能

[立即重建] [稍后提醒] [了解更多]
```

## 4. 轻量协作 UI

### 4.1 代码片段分享

**场景**: 用户想分享代码片段给同事

**设计要求**:
- 选中代码 → 右键菜单 → "分享代码片段"
- 生成分享链接（本地服务器）
- 支持语法高亮和行号
- 支持公开/私密模式

**UI 流程**:
```
1. 选中代码
2. 右键 → "分享代码片段"
3. 弹出对话框:

   分享代码片段

   文件: src/utils/parser.ts
   行数: 45-67 (23 行)

   分享模式:
   ○ 公开（任何人可访问）
   ● 私密（需要密码）

   密码: [自动生成] ████████

   [生成链接] [取消]

4. 生成链接:

   分享链接已生成

   http://localhost:3000/share/abc123

   密码: ████████
   有效期: 7 天

   [复制链接] [复制密码] [关闭]
```

### 4.2 代码行级评论

**场景**: 用户想在代码行上添加评论

**设计要求**:
- 鼠标悬停在行号上显示评论图标
- 点击图标弹出评论输入框
- 支持 Markdown 格式
- 显示评论数量徽章

**UI 元素**:
```
行号区域:
  45  [💬 2]  function parseCode(input: string) {
  46  [💬]    const ast = parser.parse(input);
  47         return ast;
  48         }

点击 [💬] 后:

  ┌─────────────────────────────────┐
  │ 评论 (2)                         │
  ├─────────────────────────────────┤
  │ @alice 2 小时前                  │
  │ 这里应该添加错误处理              │
  │                                  │
  │ @bob 1 小时前                    │
  │ 同意，建议使用 try-catch          │
  ├─────────────────────────────────┤
  │ [添加评论...]                    │
  │                                  │
  │ [发送] [取消]                    │
  └─────────────────────────────────┘
```

## 5. 设计原则

### 5.1 性能可见性
- 所有耗时操作（>500ms）必须显示进度
- 提供取消操作的能力
- 显示预估剩余时间

### 5.2 错误友好性
- 使用通俗易懂的语言
- 提供具体的解决建议
- 避免技术术语（或提供解释）

### 5.3 协作轻量化
- 无需账号系统
- 无需服务器部署
- 本地优先，数据可控

### 5.4 一致性
- 统一的图标语言
- 统一的颜色系统
- 统一的交互模式
